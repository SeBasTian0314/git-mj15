Areas de GIT

Working Directory

El area trabajo corresponde a la carpeta donde creamos el repositorio y basicamente va,os a crear, editar o eliminar archivos o subcarpetas.En muy pocas ocasiones utilizaremoa la consola y solamete necesitamos implementar el comando de GIT que inicializa el repositorio.

Staging Area (INDEX)

El area d control de cambios corresponde a un archivo dentro de la carpeta .git/ que gestiona a que archivos le realizamos cambios.Tambien se encarga de realizar la captura de codigo (SNAPSHOT) para su posterior confirmacion cuando hayamos finalizado con el trabajo correspondiente.

Repository (local/remote)

El almacen de cambios corresponden a una estructura de carpetas dentro de la carpeta em>.git/ que contiene las confirmaciones realizadas en archivos BLOG (Binary Large Object). Se utilizan principalmente como referencia para comparar, visualizar y modificar el contenido de nuestro proyecto.
